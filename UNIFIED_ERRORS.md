# Унификация ответов API (JSON и Ошибки)

В рамках третьего этапа (пункт 3 из перечня оставшихся задач) была проведена полная стандартизация ответов сервера, чтобы все данные и ошибки отдавались в едином предсказуемом JSON-формате.

## Что именно сделано

1. **Создан единый обработчик ответов (`responses.go`):**  
   - Добавлены функции-хелперы `WriteJSON` и `WriteError`. 
   - Структуры `ErrorResp` (и другие) вынесены в отдельный файл из `routes.go` для переиспользования во всех обработчиках и миддлварях.

2. **Стандартизация всех ошибок:**  
   - Ранее встречались сырые текстовые ошибки: `http.Error(w, "internal server error", 500)`.  
   - Теперь любой сбой возвращает консистентный JSON формат:
     ```json
     {
       "error": "код_ошибки_в_змейке (snake_case)",
       "code": "HTTP_ИЛИ_БИЗНЕС_КОД_КРУПНЫМ_ПРЯМЫМ_ШРИФТОМ",
       "message": "человекочитаемое сообщение"
     }
     ```
   - Заголовки ответов всегда содержат `Content-Type: application/json`.

3. **Рефакторинг всех хэндлеров:**  
   - Изменены `routes.go`, `admin.go`, `me.go` и `middleware.go`.  
   - Исчезло дублирование связок `w.WriteHeader(http.Status...)` + `json.NewEncoder(w).Encode(...)`.  
   - Были убраны лишние импорты (например, удален неиспользуемый более `"encoding/json"` в файлах `me.go` и `middleware.go`).

## Где находится код

- **Централизованные хелперы:** `apps/api/internal/handlers/responses.go`
- **Маршруты пользователей и слотов:** `apps/api/internal/handlers/routes.go`
- **Маршрут получения историй (/me/):** `apps/api/internal/handlers/me.go`
- **API Администратора:** `apps/api/internal/handlers/admin.go`
- **Миддвари авторизации:** `apps/api/internal/handlers/middleware.go`

## Как проводилось тестирование

1. **Успешная сборка (Сompatibility check):** Выполнен `docker compose build api --no-cache`, проект компилируется из-за полной совместимости возвращаемых структур.
2. **Работоспособность (Blackbox API check):** Container запущен локально, и проведены тесты к API:
   - Запросы к `/slots` успешно возвращают `{"success": true, "data": [...]}`.
   - Запросы к защищенным ручкам (например, `POST /admin/slots` с некорректным или отсутствующим JSON) вместо "падения" отдают красивый JSON вроде: `{"error":"missing_fields","code":"VALIDATION_ERROR","message":"Missing required text fields"}` со статусом 422.
3. Проверка того, что **Двойная оплата** (из предыдущих этапов) и **Middlewares** не "сломались": 
   - Структуры и логика не менялись, изменилась лишь обертка `WriteError` перед выходом из функции.
   - Миддлварь `X-Admin-Token` также корректно отбивает нарушителей 401-м статусом в JSON формате.
