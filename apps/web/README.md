# Zal Web PWA (Frontend)

Это PWA клиентское приложение (React + TypeScript + Vite) для MVP проекта Zal — сервиса поиска и бронирования спортивных слотов.
Оно разработано в стиле **Neo-Brutalism** (яркие цвета, выраженные черные границы, жесткие тени).

## Что было реализовано в рамках последних обновлений

1. **Каталог с умными фильтрами:** 
   Добавлена фильтрация игр по датам (`С` и `ПО`), а также по виду спорта и району. 
2. **Интеграция рекламного блока:** 
   Органично добавлен рекламный баннер-информер, стилизованный под нео-брутализм, который не ломает общую композицию списка слотов.
3. **Карточка слота с правилами и ценами:** 
   Детальная страница слота теперь корректно отображает цену участия (`expected_price`), текстовые правила игры (`rules_text`), а также политику отмены бронирования в контрастном желтом блоке «ВАЖНАЯ ИНФОРМАЦИЯ».
4. **Устранение вложенного репозитория (submodule-fix):** 
   Фронтенд полностью и корректно интегрирован в основной монорепозиторий проекта для прозрачной истории коммитов (Trunk-based development).

## Основные файлы и их назначение

Все основные исходники находятся в папке `src/`:

*   **`src/pages/SlotsPage.tsx`** — Главная страница каталога. Здесь располагается форма фильтров (поиск по датам `dateFrom`, `dateTo`, спорту и району), список открытых слотов и рекламный блок.
*   **`src/pages/SlotDetailsPage.tsx`** — Страница карточки конкретной игры. Выводит детали (спортивный зал, время, цену), желтый блок с правилами слота и условиями отмены. Также содержит кнопки бронирования («Участвовать») и оплаты.
*   **`src/pages/MyParticipationsPage.tsx`** — Страница личного кабинета, где пользователь видит свои оплаченные или забронированные слоты.
*   **`src/api/client.ts`** и **`src/api/index.ts`** — HTTP-клиент, построенный поверх Axios, который отвечает за отправку запросов к нашему Go API (на `http://localhost:8080`), парсинг дат и проброс токенов авторизации из заголовков.
*   **`src/api/types.ts`** — Типизация TypeScript для структур, приходящих с бэкенда (`Slot`, `Participant`, `ApiError`).
*   **`src/components/SlotCard.tsx`** — Компонент-карточка, использующийся для отображения краткой информации о слоте в списке.

Настройки инфраструктуры фронтенда:
*   **`tailwind.config.js`** — Конфигурация TailwindCSS для стилей и цветов (Neo-Brutalism).
*   **`vite.config.ts`** — Настройки сборщика Vite (включая порт `5173` или `5174` по умолчанию).

## Как запустить и проверить

Для комфортной работы фронтенд должен взаимодействовать с запущенным локальным API-сервером и базой данных.

### 1. Поднять бэкенд и БД (в корне проекта)
В корневой папке (`zal-mvp` или `SNKI-main`) выполните запуск контейнеров с базой и API, чтобы они работали в фоне:
```bash
docker compose up -d
```
Убедитесь, что API поднялся на `http://localhost:8080`.

### 2. Запуск фронтенда
Перейдите в папку веб-приложения:
```bash
cd apps/web
```

Установите зависимости (если еще не установлены):
```bash
npm install
```

Запустите локальный сервер разработки:
```bash
npm run dev
```

### 3. Проверка функционала
*   **Каталог:** Откройте в браузере ссылку, которую вам выдал терминал (обычно `http://localhost:5173` или `http://localhost:5174`).
*   **Фильтрация:** Покликайте по датам «С» и «ПО» в фильтре и нажмите поиск. Запрос улетит на API с параметрами дат, и список отфильтруется.
*   **Детали:** Нажмите "Подробнее" на любом слоте, чтобы перейти на страницу карточки (роут `/slots/{id}`). Вы должны увидеть большой желтый блок "ВАЖНАЯ ИНФОРМАЦИЯ" с правилами участия и ценой.
*   **Бронирование:** Нажмите «Участвовать» (состояние должно поменяться), а затем на кнопку имитации оплаты. Обратите внимание на защиту от двойного списания, реализованную на бэкенде.
